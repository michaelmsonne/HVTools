Behind the Tool

The story, journey, and insights into building HVTools

💡 The Origin Story

Why HVTools exists

🎯

The Problem

As a Microsoft MVP and IT professional working with Hyper-V environments and a lot more daily, I noticed a gap in the tooling landscape. While VMware administrators had RVTools for comprehensive VM inventory and reporting, Hyper-V administrators lacked a similar solution.

Existing options were either too complex, expensive, or didn't provide the quick overview needed for daily operations and reporting.

💭

The Idea

What if there was a tool that could provide instant visibility into Hyper-V environments - just like RVTools does for VMware? A tool that's free, easy to use, and provides comprehensive data export capabilities.

HVTools was born from this idea - to give Hyper-V administrators the visibility they deserve.

🚀

The Journey

Development started as a personal project to scratch my own itch. What began as simple scripts evolved into a full Windows Forms application with support for standalone hosts, clusters, and Azure Local.

The decision to make it open source came naturally - the community deserves access to quality tools.

🎨 Design Decisions

Why certain choices were made

Why Windows Forms?

While WPF and other frameworks exist, Windows Forms was chosen for:

  • 🚀 Fast startup time
  • 📊 Excellent DataGridView for tabular data
  • 💾 Smaller application size
  • 🔧 Simpler maintenance (and easy for me to create as I have used that over a longer time)
  • 👴 Familiar to IT administrators

Why .NET 10?

The latest .NET version provides:

  • ⚡ Improved performance
  • 🔒 Enhanced security features
  • 🆕 Modern C# language features
  • 📦 Better dependency management
  • 🛠️ Long-term support

Why PowerShell Backend?

PowerShell integration enables:

  • 🔌 Native Hyper-V cmdlet access
  • 🔄 Easy updates without recompilation
  • 📝 Scriptable and extensible
  • 🌐 Remote execution capabilities
  • 👥 Familiar to administrators

Why Open Source?

Making HVTools open source because:

  • 🌍 Community benefits everyone
  • 👀 Transparency builds trust
  • 🐛 More eyes find more bugs (and fix them 😉)
  • 💡 Community contributions welcome
  • ❤️ Giving back to the community

🧗 Challenges & Solutions

Obstacles encountered and how they were overcome

🔐

Remote Authentication

Challenge: Handling credentials securely across different authentication scenarios (local, domain, workgroup, VPN).

Solution: Implemented flexible credential handling using Windows Credential APIs, supporting both current user context and explicit credentials. Added Global Secure Access and VPN compatibility for modern network scenarios.

⚡

Performance with Large Environments

Challenge: Querying environments with hundreds of VMs without freezing the UI or overwhelming the hosts.

Solution: Implemented async/await patterns, background workers for long operations, and optimized WMI queries to retrieve only necessary properties.

🔗

Cluster vs Standalone Detection

Challenge: Automatically detecting whether a host is standalone, part of a cluster, or Azure Local, and adapting queries accordingly.

Solution: Created a discovery routine that probes for cluster membership first, then falls back to standalone mode. Azure Local is detected via specific registry keys and service presence.

📊

Consistent Data Export

Challenge: Ensuring exported data (JSON, CSV, XML) is consistent and properly formatted regardless of special characters or null values.

Solution: Built a robust export engine with proper escaping, null handling, and encoding. Each format has dedicated serialization logic to ensure data integrity.

📚 Lessons Learned

Insights from the development journey

01

Start Simple, Iterate

The first version was just a script. Each iteration added value. Don't wait for perfect - ship and improve.

02

User Feedback is Gold

Real-world usage reveals issues no amount of testing catches. Embrace feedback and iterate quickly.

03

Document as You Go

Writing documentation while developing is easier than retrofitting. Future you will thank present you.

04

Error Handling Matters

In IT tools, graceful error handling and clear messages are crucial. Users need to know what went wrong and how to fix it.
